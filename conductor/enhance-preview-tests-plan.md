# 目标
强化前端 E2E 测试中的重命名预览测试 (`e2e/tests/tasks/preview.spec.ts`)，增加对匹配和未匹配文件的更严格校验，包括对 UI 中 `.matched-text`、`.unmatched-text` 类名以及对应标签内容的断言。同时，新增一个针对正则捕获组与空正则替换的高级预览测试场景。

# 涉及文件
- `e2e/tests/tasks/preview.spec.ts`: 将在此文件中修改现有测试并追加新测试用例。

# 实现步骤
1. 修改 139 移动云盘预览用例：
   - 不再只断言 `.getByText(...)`。
   - 使用 `locator('tr').filter({ hasText: '...' })` 分别定位到匹配文件 (`功夫熊猫4.mp4`) 和未匹配文件 (`readme.txt`)。
   - 对匹配文件，断言其新名字位于 `.matched-text` 内，并且标签为“匹配”。
   - 对未匹配文件，断言其原名字显示在 `.unmatched-text` 内，并且标签为“未匹配”。

2. 修改夸克网盘预览用例：
   - 使用相同的逻辑，针对 `.txt` 的正则分别断言匹配文件 (`readme.txt`) 和未匹配文件 (`功夫熊猫4.mp4`)。

3. 新增 `验证正则捕获组与空正则预览场景` 用例：
   - 模拟填充分享链接后，测试正则包含捕获组如 `(.*)\.mp4`，替换为 `$1_高清版.mp4`，并验证结果。
   - 测试空正则表达式，验证所有的文件均被匹配替换（如替换表达式为 `添加前缀_{OLDNAME}`）。

# 验证与测试
- 运行 `make e2e-test` 确保更新的重命名预览测试能够完美通过。
