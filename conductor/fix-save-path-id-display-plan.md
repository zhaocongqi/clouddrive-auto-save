# 修复保存路径显示为 ID 的 Bug (Fix Save Path Display as ID Plan)

**Goal:** 修复在选择目录后，“保存路径”输入框显示为文件夹 ID 而不是人类可读绝对路径的问题。

---

## Task 1: 恢复后端路径拼接逻辑

**Files:**

- Modify: `internal/api/folder.go`

**Changes:**

1. 修改 `getAccountFolders` 函数。
2. 在遍历子目录时，基于传入的 `parent_path` 参数，手动拼接出每个子文件夹的完整绝对路径。
3. 将拼接好的绝对路径赋值给返回对象的 `path` 字段。

**Reasoning:**
之前的修改为了解决目录树名称显示问题，不小心删除了 API 层用于构造绝对路径的逻辑。对于夸克等网盘，底层驱动不返回绝对路径，需要 API 层结合上下文自行拼接。如果 `path` 字段为空或被误设为 ID，前端 `save_path` 就会显示错误的信息。

---
