# 全局调度设置优化计划 (Global Schedule Improvement Plan)

## 背景与目标 (Objective)
当前系统设置中的“全局定时任务”存在两个用户体验问题：
1. **缺少默认值**：用户未设置时没有任何初始值，建议默认为“每天凌晨”。
2. **交互不友好**：Cron 表达式对非技术用户过于复杂，需要输入空格和星号。
目标是通过后端初始化默认值，并在前端提供直观的时间选择器（或“格子”式快速选择）来简化配置。

## 关键文件 (Key Files & Context)
- `cmd/server/main.go`: 后端启动逻辑，负责加载设置。
- `web/src/views/Settings.vue`: 前端设置页面。

## 实施步骤 (Implementation Steps)

### 第一阶段：后端默认值初始化
1. **修改 `cmd/server/main.go`**：
   在加载全局调度设置的代码块中，增加判断逻辑：如果从数据库读取的 `global_schedule_cron` 为空，则自动将其设为 `0 0 0 * * *`（每天凌晨 00:00:00）并存回数据库。
   同时，确保 `global_schedule_enabled` 的默认逻辑符合预期。

### 第二阶段：前端 UI 体验升级
1. **引入“调度模式”切换**：
   在 `Settings.vue` 中增加一个 `cronMode` 状态（默认为 `daily`）。
   - **Daily 模式**：显示一个 `el-time-picker` (格式为 `HH:mm`)，用户选择时间后自动转换为 `0 mm HH * * *` 格式。
   - **Advanced 模式**：显示现有的 Cron 字符串输入框。
2. **可视化选择（“格子”思想）**：
   提供“常用预设”按钮组（如：凌晨、中午、每小时），点击即可快速填充时间。
3. **双向同步逻辑**：
   - 监听时间选择器的变化，实时更新 Cron 字符串。
   - 监听 Cron 字符串的变化（如果匹配 `0 mm HH * * *` 模式），自动回填时间选择器。

## 验证与测试 (Verification & Testing)
1. **后端验证**：删除数据库文件或清空 `settings` 表后启动服务，验证数据库中是否自动生成了 `0 0 0 * * *`。
2. **前端验证**：
   - 进入设置页面，默认看到“每天 00:00”的时间选择器。
   - 修改时间（如 01:30），保存后刷新，验证仍显示为 01:30。
   - 切换到“高级模式”，手动修改 Cron 表达式，验证功能依然正常。
   - 检查非法 Cron 表达式的错误提示是否依然有效。
