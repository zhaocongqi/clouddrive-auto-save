# 设计规范：项目图标与首页视觉重构 (2026-05-05)

## 1. 背景与目标 (Context & Objectives)
当前项目的网页标签图标 (Favicon) 和首页图标被认为过于随意，缺乏专业感和品牌一致性。本设计的目的是通过重新设计核心视觉元素，传达“统一云盘自动转存”这一核心价值。

**目标：**
- 建立专业、现代的视觉形象。
- 强调“云端存储”和“自动化”意象。
- 优化不同尺寸下的辨识度（从 16x16 的 Favicon 到首页大图）。

## 2. 设计方案：层叠云朵 (Layered Cloud)
选择方案 A，利用多层渐变色块模拟云朵的厚度和层次感。

### 2.1 核心调色盘
- **主色 (Sky Blue)**: `#4facfe` (渐变起始点)
- **亮色 (Cyan)**: `#00f2fe` (渐变结束点)
- **浅背景色**: `#f0f9ff` (用于 UI 容器背景)
- **强调色 (Deep Blue)**: 用于阴影或深度感。

### 2.2 视觉元素
- **形态**: 圆润、饱满的云朵轮廓。
- **层次**: 内部包含一个象征“数据核心”或“同步状态”的圆形元素。
- **风格**: 微渐变、轻量化阴影。

## 3. 应用规格 (Application Specs)

### 3.1 网页标签图标 (Favicon)
- **路径**: `web/public/favicon.svg`
- **尺寸**: 矢量 SVG，适配 16x16, 32x32。
- **实现**: 简化版云朵轮廓，使用单层或双层渐变以保证小尺寸下的清晰度。

### 3.2 首页/仪表盘图标 (Hero Icon)
- **路径**: 建议替换 `web/src/assets/hero.png` 为新的 SVG 实现或 Vue 组件。
- **样式**: 完整版层叠设计，带有柔和的外阴影 (`box-shadow: 0 10px 20px rgba(79, 172, 254, 0.3)`)。

### 3.3 交互动态 (Interaction)
- **转存进行中**: 云朵中心的“核心”圆形产生呼吸灯效果 (`opacity` 从 0.4 到 1.0)。
- **成功反馈**: 云朵边缘产生一次性的流光扩散效果。

## 4. 实施要点 (Implementation Points)
1. **SVG 代码优化**: 使用纯 SVG 实现渐变和形状，确保在不同分辨率下不失真。
2. **替换静态资源**: 更新 `web/public/favicon.svg`。
3. **前端代码修改**: 在 Dashboard 首页引用新的视觉元素，移除旧的 `hero.png` 依赖（如果适用）。

## 5. 自我审查 (Spec Self-Review)
- **占位符**: 无。
- **一致性**: 色值和风格在全站统一。
- **范围**: 仅限于图标和首页核心视觉重构，不涉及整体 UI 框架大改。
- **模糊性**: 明确了呼吸灯动画的具体实现方式。
